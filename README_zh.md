# 🚀 LiteUI Studio

LiteUI Studio 是一个开箱即用、完全独立的本地 AI 视听创作工作站。它以极其硬核的底层显存调度机制，将庞杂的节点式连线（ComfyUI）封装成了对人类直觉最友好的现代化交互界面。无需折腾复杂的代码环境，只需双击运行，即可在本地流畅驱动目前全球最前沿的图像与视频生成模型。

## ✨ 核心亮点 (Features)

- 🎮 **真正的开箱即用**：内置经过极致调优的便携版 Python 环境（感谢[秋葉aaaki](https://space.bilibili.com/12566101/dynamic)大佬），**无需配置 Conda、无需敲击 `pip` 命令**，解压后双击 `.bat` 即可启动。
- 🎬 **大满贯旗舰引擎**：
  - **Flux2**：支持文生图、图生图，搭配 LoRA 实现电影级画质。
  - **Wan 2.2**：图生视频引擎，支持轻量无声模式与内置 MMAudio 环境音轨合成模式。
  - **LTX-2.3**：原生声像画同轨引擎，支持“导演时间轴”提示词（如 `0-1秒: 摘帽子, 1-8秒: 说话`），实现精准对口型。
- 🔄 **无缝流转工作流**：在“图像页”抽出的完美盲盒，点击“一键发送”，瞬间无缝流转至“视频页”作为初始帧，无需反复保存上传。
- 🔍 **Metadata读取器 (Metadata Inspector)**：支持“即抛即得”。将生成的原图或视频拖入提取器，瞬间还原当时的提示词、种子和参数配方。

## 💻 硬件要求 (Requirements)

- **硬盘空间**：88 GB（包含约75 GB的预装模型文件）
- **操作系统**: Windows 10 / 11
- **显卡**: **NVIDIA** GPU，显存 $\ge$ 8GB（强推 RTX 4060 / 3060 等消费级甜品卡）。6G显存请在合理调整生成参数后使用。
- **内存**: 推荐16~32GB（因为大模型置换需要占用系统内存，建议开启至少 20GB 的 Windows 虚拟内存）。若显存较低，请在合理调整生成参数后使用。

## 🛠️ 安装与使用 (Getting Started)

由于项目包含了完整的运行库但未包含动辄几十 GB 的模型权重，请在新电脑上按以下步骤进行初始化：

### 1. 准备 FFmpeg（核心组件）
本项目依赖 FFmpeg 进行极速的无损音视频混流与元数据注入。
- 请下载 Windows 版的 `ffmpeg.exe` 和 `ffprobe.exe`，并将它们**直接放在本项目的根目录下**（与 `start.bat` 同级）。

### 2. 一键启动
- 双击根目录下的 **`start.bat`**。
- 喝口水，等待终端显示“引擎启动成功”，现代化的控制台网页将自动在浏览器中弹出，尽情创作吧！

### 3. 添加更多模型和LoRA
- **注意**：**本模型仅支持GGUF量化模型。**添加的模型必须与预设模型的架构一致。如文生图模型只能是基于FLUX.2-klein-9B的其他微调版本（4B模型将会失效）。
- 1. **[Hugging Face](https://huggingface.co/)**
   * 全球最大的 AI 开源社区与学术界基石。
   * **推荐下载**：Flux2, Wan2.2, LTX-2.3 的量化模型和其他微调版本。
- 2. **[Civitai](https://civitai.com/)**
   * 全球最活跃的生成式 AI 创作者社区。
   * **推荐下载**：各类特定画风、特定人物或材质的 LoRA 模型。
   * *注：作为一个极度自由的创作者 UGC 社区，该平台包含海量由网民上传的生成内容。请在浏览和下载时自行甄别，并严格遵守本项目的免责声明与当地法律法规。*

## 💡 使用小贴士 (Tips)

- **硬件报警拦截**：如果你参数设置过高导致底层 OOM，网页右上角会立刻弹出清晰的中文红色报错，同时底层会自动清空显存，你无需重启软件即可重新调整参数生成。
- **强制急刹车**：生成途中如果发现提示词写错了，直接点击界面上的 `[🛑 打断]` 按钮，任务会瞬间终止并退回所有占用的显存。

 ## 🙋 常问问题 (Q&A)

- **Q: LiteUI Studio 是付费项目吗？**
- A: LiteUI Studio 完全开源免费，且没有任何隐藏的付费内容。GitHub 官方发布与维护地址为：https://github.com/FJWRnoArina/LiteUI-Studio/

- **Q: LiteUI Studio 与主流的 SD-webui 和 ComfyUI 有什么区别？**
- A: SD-webui 和 ComfyUI 对于入门用户都存在较高的学习门槛。
- 使用 SD-webui 需要用户手动配置底层模型、CLIP、VAE、负向提示词、引导强度、采样器、步数、CFG Scale、Upscaler、Denoising Strength、高清修复、VAE Tile等参数，且其显存管理优化较弱。
- ComfyUI 虽然拥有顶级的显存管理，但入门门槛极高，需要用户熟知各个节点的作用、模块化管理和连线逻辑。
LiteUI Studio 以 ComfyUI 强大的 API 作为底层引擎，继承了其优秀的显存管理机制；同时在前端用现代化的 WebUI 封装了复杂的文生图、图生图和多模态视频生成工作流，极大简化了用户的操作体验。

**Q: LiteUI Studio 是基于 ComfyUI 架构的吗？**
A: 是的。项目 `core/workflows` 文件夹中的 JSON 文件都是可以直接在原生 ComfyUI 中导入并打开的工作流。实际上，在 LiteUI Studio 运行时，你可以随时在浏览器访问 `http://127.0.0.1:8188/` 来直接打开并监控底层的 ComfyUI 连线界面。

**Q: 作为开发者，为什么不自主设计一套 Python 模型推理架构？**
A: 这正是 LiteUI Studio 在最初设计时踩过的坑。整合各大前沿开源项目最大的痛点在于**“兼容性地狱”**。在手写管线时，我们发现 `transformers`、`xformers`、模型 pipeline 库、量化计算库以及底层工具链（`torch`, `numpy` 甚至 `python` 版本）之间的兼容性极其糟糕；经常遇到版本对不齐，甚至底层出现数层空映射的情况，修复成本极高。ComfyUI 社区已经极其完美地解决了庞杂模型管线与量化库之间的兼容性问题，这是我们选择它作为坚实底座的最重要原因。

**Q: LiteUI Studio 支持生成任何内容吗？**
A: 生成的内容取决于您选择的底层模型以及输入的提示词。请务必在严格遵守本项目的【免责声明】与您当地法律法规的前提下使用本软件。

**Q: 界面中有一些我看不懂的专业选项或参数怎么办？**
A: 建议将不理解的参数名称或报错截图，直接发送给主流 AI 助手（如 豆包、ChatGPT、Gemini 等）或使用搜索引擎进行查询，以获取最详尽的解释。

**Q: 运行时遇到了我看不懂或无法修复的报错代码。**
A: 
1. 终端中的一些橙色/黄色警告通常不影响正常运行，可以忽略。
2. 如果导致生成中断，请先将报错内容交给 AI 助手（如 豆包、ChatGPT、Gemini 等）进行分析。
3. 尝试重启 LiteUI Studio 释放可能卡死的显存。
4. 如果依然无法解决，欢迎在 GitHub 仓库的 Issues 页面提出。

**Q: 为什么软件无法加载我自己下载的模型？**
A: 为保证在 8GB 显存下的极限运行稳定性，LiteUI Studio 目前进行了硬编码路由，**仅支持以下三大模型的 GGUF 量化版本**（及其派生的微调模型）：`FLUX.2-klein-9B`、`Wan2.2-I2V-A14B`、`LTX-2.3`。截至 2026 年 7 月，它们是在有限本地显存下能跑出的最高质量模型。如需使用其他架构模型，请使用原生的 SD-webui 或 ComfyUI。

**Q: 为什么无法加载我提供的 LoRA？**
A: 请确保您下载的 LoRA 所适配的底层架构与当前标签页的模型（Flux2 / Wan2.2 / LTX-2.3）完全一致。注意：用于生成图片（Flux）的 LoRA 绝对不能用于生成视频（Wan/LTX），反之亦然。

**Q: 我不知道怎么写提示词 (Prompt)。**
A: 主流的 AI 助手（豆包、ChatGPT、Gemini 等）是极佳的提示词代写工具。您可以告诉它们您想要的画面，让其翻译为英文提示词，复制粘贴即可。

**Q: 我想追溯一张几个月前生成的、质量很好的图片/视频的参数配方。**
A: 将该原图或原视频直接拖入软件界面的 **“🔍 魔法放大镜 (Metadata Inspector)”** 标签页即可瞬间解析。

**Q: 为什么生成的质量似乎没有网上演示的那么高？**
A: 物理硬件的妥协。为适配 8GB 低显存，LiteUI Studio 预装并驱动的是经过极度压缩的 GGUF 量化版模型（如 Q4 / Q2 等），这在数学上不可避免地会损失部分生成质量。您可以尝试：多次随机生成（抽卡）、优化提示词细节，或适当提高采样步数 (Steps)。

**Q: 同样是视频生成，Wan2.2 和 LTX-2.3 有什么区别？**
A: 
- **Wan2.2** 基础模型本身不支持声音。在 LiteUI Studio 中，我们为其额外挂载了 MMAudio 模块来实现极其逼真的环境音效，但它不支持生成人类语音台词与对口型。
- **LTX-2.3** 是原生多模态联合架构。它不仅支持环境音，还原生支持设定时间轴的解说台词，并能自动完成人物画面的精准对口型 (Lip-sync)。
